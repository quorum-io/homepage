Technology

Content-addressed governance over authenticated mesh networks

Weighted Quorum

Consensus requires 67% of weighted signals. Votes decay exponentially with a 5-minute half-life. One vote per DID, no self-voting.

threshold = ⌈(n + 1) × 0.67 × 1000⌉

Content Addressing

Every expression has a unique CID. SHA-256 hash includes 256-bit random nonce and timestamp. Automatic deduplication.

cid = sha256(data || random || ts)

Post-Quantum Signatures

Dilithium3 lattice-based signatures on all messages. Resistant to quantum attacks. Protects against harvest-now-decrypt-later.

sig = dilithium3.sign(sk, msg)

S-Expression Protocol

Homoiconic format for proposals, votes, and messages. Signed envelopes wrap all expressions. Parse once, verify everywhere.

(signed pk sig (propose "..."))

Principles

Security model and design constraints

01

Human Approval

Peer connections require elaboration and manual approval. Sybil resistance through human verification, not proof-of-work or stake.

02

No Self-Voting

Proposers cannot vote on their own proposals. Trust scores weight votes. Quality participation increases influence.

03

Defense in Depth

Rate limiting, nonce tracking, signature verification, size limits. Each layer assumes the others have failed.

04

Open Discovery

Probe any node for pools and peers without authentication. Bootstrap the network from a single known address.

05

Derived State

Store expressions, compute everything else. The expression log is the source of truth. State is a materialized view.

06

Quantum Resistant

Dilithium3 signatures from day one. No migration path required when quantum computers arrive.

Network

Pool-based trust domains with authenticated mesh topology

Quorum

Genesis Pools

Three hardcoded pools bootstrap the network. New pools are proposed and voted into existence by authenticated participants.

Pool Isolation

Pools define trust boundaries. Nodes only connect to peers in the same pool. Different organizations use the same protocol without interference.

Trust Scores

Scores update with each interaction: score = score × 0.7 + quality × 0.3. Minimum score of 0.4 required to propose.
